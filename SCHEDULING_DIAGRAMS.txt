2. EVENT TIMING CALCULATION
================================================================================

MIDI Data → Scheduled Time Formula:

    MIDI Ticks (960)
          │
          ├─→ Divide by 480 (ticks/beat)
          │   960 ÷ 480 = 2 beats
          │
          ├─→ Multiply by (60 / BPM)
          │   2 × (60 / 120) = 1.0 second
          │
          ├─→ Add region start time
          │   1.0 + 0.3 = 1.3 seconds (relative to transport)
          │
          ├─→ Add countdown offset (if enabled)
          │   1.3 + 0.0 = 1.3 seconds
          │
          └─→ Add loop offset (if repeating)
              1.3 + 0.0 = 1.3 seconds FINAL TIME


BATCHING STRATEGY:
┌────────────────┐
│ absoluteTime   │  1.3000 seconds
└────────────────┘
          │
          ├─→ Math.round(1.3000 * 1000) = 1300
          │   Math.round(1300) / 1000 = 1.300
          │
          └─→ Group by this timeKey
              ├─ All events at 1.300s go together
              ├─ Prevents sequential callback delays
              └─ Usually 3-4 instruments per batch


3. FRAME-BASED TIMING (48kHz Sample Rate)
================================================================================

TIME ↔ FRAME CONVERSION:

    frames = seconds × 48000
    
    0.5s  → 24,000 frames
    1.0s  → 48,000 frames
    1.3s  → 62,400 frames
    2.0s  → 96,000 frames

LOOK-AHEAD IN FRAMES:

    150ms look-ahead = 150 × 48 = 7,200 frames

    ┌─────────────────────────────────────┐
    │ At scheduling (t=1.85s):            │
    │ scheduleFrame = 88,800 frames       │
    │                                     │
    │ Target play (t=2.0s):               │
    │ targetFrame = 96,000 frames         │
    │                                     │
    │ Look-ahead distance:                │
    │ 96,000 - 88,800 = 7,200 frames      │
    │ 7,200 ÷ 48 = 150ms ✓                │
    └─────────────────────────────────────┘

SCHEDULING EVENT VISUALIZATION:

    Frame Timeline (48kHz):
    ─────────────────────────────────────────
    0       24000    48000    72000    96000   frames
    0s      0.5s     1.0s     1.5s     2.0s    seconds
                                ↑
                          Event should play here
                          (targetFrame = 96000)

    At scheduler check (t=1.85s = 88800 frames):
    
    ──────────────────────╥──────────────────────
    ...    88800   90600  99000  103200...     frames
    ...    1.85s   1.887s 2.06s  2.15s...     seconds
           ↑              ↑
        Check here    Schedule until here
        (current)     (current + 150ms)

6. TEMPO CHANGE COMPENSATION
================================================================================

SCENARIO: BPM changes from 120 → 140 during playback

Timeline:
┌─────────────────────────────────────────────────────┐
│                                                     │
│ At t=2.5s: BPM = 120, accumulatedBeats = 0         │
│ ├─ Elapsed: 2.5s                                   │
│ ├─ Beats: 2.5 × (120/60) = 5 beats                 │
│ └─ Store: accumulatedBeats = 5                     │
│                                                     │
│     [TEMPO CHANGE: 120 → 140 BPM]                  │
│                                                     │
│ At t=2.6s: BPM = 140, accumulatedBeats = 5        │
│ ├─ Elapsed since change: 0.1s                      │
│ ├─ Beats since change: 0.1 × (140/60) = 0.233     │
│ ├─ Total beats: 5 + 0.233 = 5.233 beats           │
│ └─ Position: 5.233 / (140/60) = 2.244s            │
│                                                     │
└─────────────────────────────────────────────────────┘

RESULT:
    Without compensation:
    └─ Position would jump from 2.5s to 2.6s (wrong!)

    With compensation:
    └─ Position smoothly transitions
    └─ No jitter or visual glitches
